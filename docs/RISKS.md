# Риски и способы их решения

## Данные и контент
| Риск | Влияние | Решение |
|---|---|---|
| Неполные/неуточнённые данные (год, телефон, регион Gastro-Ferma; ассортимент ряда хозяйств) | Карточки выглядят «пусто», доверие ниже | Поле `notes` фиксирует, что уточнить; опциональные блоки скрываются; дозаполнять по мере отзвона. |
| Фото только ссылками на Я.Диск, не загружены | Карточки без фото | emoji-fallback не ломает вёрстку; задача — скачать, оптимизировать, разложить в `assets/images/<id>/`. |
| Smart Farm — Владимирская обл., а каталог «Нижегородской» | Несоответствие позиционированию | По решению заказчика оставлен; в `notes` зафиксировано. При желании — добавить признак «партнёр региона». |
| Ошибка в JSON (запятая, неверный `category`) | Каталог не грузится / фермер пропадает | Валидация перед коммитом (команда в README); CI-проверка JSON; `category` сверять со `slug`. |
| Дубли/смена `id` | Битые ссылки на страницы фермеров | `id` неизменен после публикации; правило в `CLAUDE.md`. |

## Технические
| Риск | Влияние | Решение |
|---|---|---|
| `fetch` не работает с `file://` | «Не удалось загрузить данные» при открытии файла напрямую | Локально — через `http.server`; на Tilda — инлайн `window.FARMERS_DATA` (без fetch). |
| Ограничения T123-блока Tilda (размер кода) | Сложно вставить весь код | Стили/данные — в head сайта; данные можно тянуть с CDN (Pages/jsDelivr). |
| SEO при клиентском рендере | Слабая индексация | JSON-LD, ЧПУ, при росте — статическая генерация (см. `SEO.md`, `SCALING.md`). |
| Внешняя зависимость (Google Fonts / Я.Диск / Sheets) недоступна | Деградация шрифтов/данных | Самохостинг шрифтов (fallback уже есть); данные — статический JSON, а не рантайм-зависимость от Google. |
| XSS через контент | Поломка/инъекция | Весь вывод экранируется (`esc`); внешние ссылки `rel="noopener"`. |
| Рост числа карточек → тормоза | Медленный каталог | Пагинация/виртуализация, статическая генерация (`SCALING.md`). |

## Процессные
| Риск | Влияние | Решение |
|---|---|---|
| Контент-менеджер боится Git | Каталог не наполняется | Этап 2: Google Sheets как редактор (`DATA-SOURCE.md`). |
| Расхождение реестра (Excel) и сайта | Устаревшие данные | Один скрипт-конвертер `xlsx→json`; единый источник правды — таблица/JSON. |
| Потеря исходников фото | Невозможно перегенерировать | Хранить оригиналы отдельно (Я.Диск/облако), ссылки — в `photosSource`. |
| Нет владельца обновлений | Каталог «протухает» | Назначить ответственного в Ассоциации; регламент обновления (`DEPLOYMENT.md`). |

## Юридические/прочие
| Риск | Влияние | Решение |
|---|---|---|
| Публикация контактов/фото без согласия | Претензии производителей | Согласие на размещение при отзвоне (статус `called`); хранить подтверждение. |
| Авторские права на фото | Претензии | Использовать только фото, предоставленные производителем/снятые для каталога. |
