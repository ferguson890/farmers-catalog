# Производительность

Цель — быстрая загрузка каталога даже при росте числа производителей. Главный «вес» каталога — фотографии, поэтому основной фокус на них.

## Изображения (приоритет №1)
- **Формат:** WebP (AVIF — опционально). JPG как запасной вариант.
- **Размеры:** обложка ~1200×800, не больше реально отображаемого. Не грузить «оригиналы из камеры».
- **Вес:** обложка ≤ 200 КБ, фото галереи ≤ 150 КБ.
- **Ленивая загрузка:** `loading="lazy"` на всех фото каталога/галереи (уже стоит) — грузятся по мере скролла.
- **Адаптивные картинки:** при необходимости `srcset`/`<picture>` для мобильных (отдавать меньший размер).
- **Размеры заданы CSS** (фикс. высота + `object-fit`) — нет «дёрганья» layout (CLS).
- Пакетная оптимизация: `squoosh`, `sharp`, `imagemin` или онлайн-сжатие.

## Запросы и ресурсы
- **Минимум файлов:** 1 CSS + 1 JS на страницу + 1 JSON. Иконки — инлайн SVG (0 доп. запросов).
- **На Tilda** данные встроены инлайном (`window.FARMERS_DATA`) — даже JSON-запрос не нужен.
- **Шрифты:** `preconnect` к Google Fonts (уже есть); подгружать только нужные начертания (300/400/600/700). При желании — самохостинг шрифтов с `font-display: swap`.
- **Кэширование:** статика хорошо кэшируется CDN (Tilda/Pages/Netlify это делают сами).

## JS
- Чистый ES5, без фреймворков и рантайм-зависимостей — минимальный парс/исполнение.
- Рендер строкой через `innerHTML` один раз на список — быстро для десятков-сотен карточек.
- Фильтр/поиск работают в памяти, без сетевых запросов.

## Когда записей станет много (сотни+)
- Пагинация или «показать ещё» / бесконечная прокрутка порциями.
- Виртуализация списка (рендерить только видимые карточки).
- Дебаунс поля поиска (сейчас не нужен — данных мало).
- Статическая генерация страниц фермеров (см. `SCALING.md`) — лучший рендер и SEO.

## Метрики и проверка
- Цель Lighthouse (mobile): Performance ≥ 90, CLS ≈ 0, LCP < 2.5s.
- LCP-элемент — обложка/герой: держать его лёгким.
- Проверять на 3G-throttling в DevTools.

## Чек-лист
- [ ] Все фото в WebP и сжаты до лимитов.
- [ ] `loading="lazy"` на нефолдовых фото.
- [ ] Размеры изображений зафиксированы (нет CLS).
- [ ] Только нужные начертания шрифтов.
- [ ] Lighthouse mobile ≥ 90.
