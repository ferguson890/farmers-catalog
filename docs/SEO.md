# SEO

Каталог производителей — потенциальный источник органического трафика («фермерский сыр Нижний Новгород», «городецкий пряник купить» и т.п.). Поэтому SEO заложено в архитектуру (отдельные URL у фермеров, серверо-независимая разметка).

## Базовое (сделано в шаблонах)
- Уникальный `<title>` и `<meta name="description">` на каждой странице. На странице фермера они проставляются из данных (`farmer.js`).
- `lang="ru"`, корректная кодировка, viewport.
- Семантические теги: `<header>`, `<main>`, `<footer>`, `<article>`, один `<h1>` на страницу.
- `og:`-теги для красивых превью при шеринге.
- `<link rel="canonical">` — указать финальный домен.

## Что добавить при публикации

### 1. Структурированные данные (Schema.org)
На странице фермера — JSON-LD `LocalBusiness`/`Organization`:
```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "LocalBusiness",
  "name": "Ресторан-сыроварня «Брынза»",
  "address": { "@type": "PostalAddress", "addressLocality": "Нижний Новгород", "streetAddress": "ул. Родионова, 187в" },
  "telephone": "+78314159119",
  "url": "https://nireal.ru/farmer/brynza"
}
</script>
```
В каталоге — `ItemList` с производителями. `farmer.js` может генерировать JSON-LD из тех же данных.

### 2. sitemap.xml и robots.txt
- `sitemap.xml` со всеми URL фермеров + страницей каталога. При статической генерации (см. `SCALING.md`) генерируется автоматически.
- `robots.txt` с ссылкой на sitemap.

### 3. Человекочитаемые URL
- Лучше `/farmer/brynza`, чем `/farmer?id=brynza`. На Tilda — отдельные страницы (Путь B в `TILDA.md`); на своём хостинге — статическая генерация или rewrite.

### 4. Контент
- Уникальные описания (у нас уже развёрнутые, не дубли).
- Осмысленные `alt` у фото (название хозяйства + продукт).
- Внутренняя перелинковка: каталог ↔ страницы, связь с разделами nireal.ru.

## Особенности SPA-рендера
Контент рисуется JS из JSON. Google это индексирует, но:
- для гарантии — рассмотреть пререндер/статическую генерацию HTML (см. `SCALING.md`), чтобы контент был в исходном HTML;
- на Tilda карточки можно частично продублировать статически для индексации.
