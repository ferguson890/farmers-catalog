# Реализация и публикация на Tilda

## Рекомендуемый способ
**Отдельная страница Tilda + один HTML-блок T123 со всем кодом каталога**, плюс при необходимости Zero Block для шапки/подвала, если хотите управлять ими в Tilda. Данные — инлайном через `window.FARMERS_DATA` (без `fetch`, без CORS).

Почему так: каталог — это единое интерактивное приложение (фильтры, поиск, рендер). Собирать его из стандартных блоков Tilda нельзя без потери логики. T123-блок даёт полный контроль над HTML/CSS/JS и при этом живёт внутри Tilda-страницы (общая шапка/футер сайта, SEO-настройки Tilda).

## Пошагово

### 1. Страница каталога
1. Создать страницу, напр. `/catalog`.
2. Добавить блок **T123 «HTML-код»**.
3. Вставить разметку каталога (содержимое `index.html` без `<head>`-обвязки: блоки HERO, FILTER, `#grid`).
4. Стили: содержимое `styles.css` обернуть в `<style>…</style>` и положить в тот же блок (или в «Настройки сайта → Ещё → HTML-код в head»).
5. Данные: вместо `fetch` встроить инлайн:
   ```html
   <script>window.FARMERS_DATA = { /* содержимое farmers.json */ };</script>
   <script> /* содержимое catalog.js */ </script>
   ```
   `catalog.js` уже поддерживает `window.FARMERS_DATA` — если он задан, `fetch` не вызывается.
6. Шрифты Playfair Display + Lato подключить в head (ссылка из `index.html`).

### 2. Страницы производителей — два пути

**Путь A (быстрый, 1 страница на всех):** одна Tilda-страница `/farmer` с блоком T123, который содержит `farmer.js` + те же инлайн-данные. Открывается как `/farmer?id=<id>`. Карточки в каталоге уже ведут на `farmer.html?id=` — поменять базовый путь в `catalog.js` (`FARMER_PAGE`) на `/farmer`.
- ➕ Один блок, не нужно плодить страницы.
- ➖ Хуже SEO: у всех фермеров один URL Tilda (мета задаётся скриптом, но Tilda-индексация слабее).

**Путь B (лучше для SEO, рекомендуется при росте):** отдельная Tilda-страница на каждого фермера (`/farmer/brynza` и т.д.), на каждой — блок с данными конкретного фермера. Больше ручной работы, но настоящие URL и мета Tilda. Можно генерировать полуавтоматически.

> Для старта — Путь A. Когда каталог станет важен для поисковой выдачи — перейти на Путь B (или вынести на отдельный хостинг, см. `DEPLOYMENT.md`).

### 3. Обновление данных
- Контент правится в `farmers.json` (репо) → копируется в инлайн `window.FARMERS_DATA` в блоке T123.
- Чтобы не копировать вручную, можно держать `farmers.json` на CDN (GitHub Pages/jsDelivr) и оставить в коде `fetch` на абсолютный URL — тогда Tilda тянет данные сама. Минус — внешняя зависимость и CORS-настройка (GitHub Pages отдаёт CORS-заголовки, работает).

## Чек-лист публикации на Tilda
- [ ] Подключены шрифты в head.
- [ ] Стили в `<style>` или в head сайта.
- [ ] `window.FARMERS_DATA` задан до подключения `catalog.js`.
- [ ] `FARMER_PAGE` указывает на реальный путь страницы фермера.
- [ ] Проверены desktop/tablet/mobile в превью Tilda.
- [ ] Заданы Title/Description страницы в настройках Tilda.
- [ ] Страница опубликована и добавлена в меню сайта.
