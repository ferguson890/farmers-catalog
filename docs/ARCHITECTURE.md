# Архитектура

## Обзор
Статический data-driven каталог. Нет бэкенда, нет сборки, нет фреймворков. Вся динамика — на клиенте: JS читает `data/farmers.json` и рендерит интерфейс. Это даёт максимальную простоту хостинга (Tilda / GitHub Pages / любой CDN) и поддержки.

```
┌─────────────────┐      fetch       ┌──────────────────┐
│  data/farmers.json  │  ◀───────────  │  catalog.js / farmer.js │
└─────────────────┘                   └──────────┬───────┘
        ▲                                         │ render
        │ редактирует контент-менеджер            ▼
        │                              ┌──────────────────┐
   (Git / Sheets-экспорт)             │ index.html / farmer.html │
                                       └──────────────────┘
```

## Слои
1. **Данные** — `data/farmers.json`: `meta`, `categories[]`, `farmers[]`. Единственный источник правды.
2. **Логика** — `src/js/`:
   - `catalog.js`: загрузка → построение табов категорий → фильтрация (категория + поиск) → рендер карточек.
   - `farmer.js`: чтение `?id=` → поиск фермера → рендер страницы.
3. **Представление** — `index.html`, `farmer.html`, `src/css/styles.css`. Палитра и типографика централизованы в `:root`.

## Потоки данных
- **Каталог:** `index.html` → `catalog.js` → `fetch(data/farmers.json)` → рендер сетки. Фильтр/поиск работают в памяти, без перезагрузки.
- **Страница:** клик по карточке → `farmer.html?id=<id>` → `farmer.js` → тот же JSON → рендер.
- **Tilda-режим:** данные можно встроить инлайном через `window.FARMERS_DATA` (см. `TILDA.md`), тогда `fetch` не нужен.

## Ключевые архитектурные решения
| Решение | Почему |
|---|---|
| Статика без сборки | Хостинг на Tilda, нулевой порог поддержки, нет CI для билда. |
| Один JSON как источник | Контент-менеджер добавляет фермера без касания кода. |
| ES5-совместимый IIFE-JS | Безопасно встраивается в HTML-блок Tilda, работает в старых вебвью. |
| Отдельная страница вместо только модалки | SEO (свой URL/`<title>` на каждого фермера), шеринг ссылок. |
| emoji-fallback для фото | Каталог не «ломается», пока фото не загружены. |
| Экранирование вывода (`esc`) | Защита от поломки разметки и XSS при вставке контента. |

## Расширение
- Новый фермер/категория — правка JSON (см. `SCALING.md`).
- Рост до сотен записей — переход на ленивую отрисовку/пагинацию или генерацию статических страниц (см. `SCALING.md`).
- Смена источника на Google Sheets/Airtable — заменить шаг загрузки данных (см. `DATA-SOURCE.md`).
