# Пошаговый план реализации

От текущего состояния до запуска. Этапы можно вести параллельно, где не указано иное.

## Этап 0 — Готово ✅
- [x] Разобран реестр `Реестр_Фермеров_Итог.xlsx` (6 производителей, 4 категории).
- [x] Спроектирована модель данных (`DATA-MODEL.md`).
- [x] Собран `data/farmers.json` с реальными данными.
- [x] Реализован каталог (`index.html` + `catalog.js` + `styles.css`).
- [x] Реализована страница производителя (`farmer.html` + `farmer.js`).
- [x] Подготовлена документация (`docs/`), `README.md`, `CLAUDE.md`.

## Этап 1 — Контент и фото
1. Скачать фото с Я.Диска (ссылки в `photosSource`), оптимизировать (`PERFORMANCE.md`), разложить в `assets/images/<id>/cover.jpg`.
2. Дозаполнить пробелы по `notes`: год основания и регион Gastro-Ferma, ассортимент и фото «Городецкие мотивы»/«Пряник из Городца»/«Сливочная улица».
3. Получить согласие производителей на публикацию (если ещё нет).

## Этап 2 — Проверка и QA
1. Локальный прогон: `python -m http.server 8000`, проверить каталог, фильтры, поиск, все страницы фермеров.
2. Адаптивность: desktop/tablet/mobile (`RESPONSIVE.md`).
3. Lighthouse mobile ≥ 90 (`PERFORMANCE.md`).
4. Валидация JSON, проверка категорий.

## Этап 3 — Публикация на Tilda
1. Создать страницу `/catalog`, вставить код в T123 (`TILDA.md`).
2. Решить путь страниц фермеров: A (единая `/farmer?id=`) для старта или B (отдельные страницы) для SEO.
3. Подключить шрифты и стили в head сайта.
4. Задать Title/Description, добавить в меню, опубликовать.

## Этап 4 — SEO и аналитика
1. JSON-LD на страницах фермеров, `sitemap.xml`, `robots.txt` (`SEO.md`).
2. Подключить Яндекс.Метрику/аналитику Tilda.
3. Зарегистрировать в Яндекс.Вебмастер и Google Search Console.

## Этап 5 — Передача и регламент
1. Назначить ответственного за наполнение.
2. При необходимости — настроить Google Sheets как редактор + автоэкспорт в JSON (`DATA-SOURCE.md`).
3. Зафиксировать регламент обновления (`DEPLOYMENT.md`).

## Дальнейшее развитие (по триггерам — `SCALING.md`)
- Переход на Google Sheets, когда наполняет не-разработчик.
- Статическая генерация страниц при >100 записей или приоритете SEO.
- Поисковый индекс/виртуализация при сотнях записей.
