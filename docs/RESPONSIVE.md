# Адаптивная вёрстка

## Подход
Mobile-friendly через прогрессивное упрощение макета на двух брейкпоинтах. Базовый макет — десктопный, на узких экранах сетка схлопывается, второстепенные элементы скрываются. Всё на CSS, без JS.

## Брейкпоинты
| Диапазон | Устройство | Что меняется |
|---|---|---|
| > 1000px | Desktop | Сетка карточек в 3 колонки, герой со статистикой. |
| 681–1000px | Tablet | Сетка в 2 колонки (`@media(max-width:1000px)`). |
| ≤ 680px | Mobile | 1 колонка; скрыты меню-ссылки и блок статистики героя; паддинги уменьшены; поиск на всю ширину; галерея фермера в 2 колонки; футер в столбец. |

## Ключевые приёмы
- **Сетка:** `grid-template-columns: repeat(3,1fr)` → `1fr 1fr` → `1fr`.
- **Тайповая шкала:** заголовки на `clamp()` (`clamp(32px,4vw,50px)` в герое, `clamp(26px,3.4vw,38px)` на странице) — плавно масштабируются без отдельных правил.
- **Фото:** `object-fit: cover` + фиксированная высота — карточки одинаковы при любых пропорциях исходника.
- **Горизонтальный скролл фильтров:** на узких экранах вкладки категорий прокручиваются по горизонтали (скрытый скроллбар), не переносясь.
- **Тач-цели:** телефоны/ссылки — крупные, легко попасть пальцем.
- **Sticky-навигация и фильтр** остаются доступными при скролле (`position: sticky`).

## Что проверять при правках
1. Каталог в 3/2/1 колонки на соответствующих ширинах.
2. Шапка на мобильном: лого + контакты, без наезда.
3. Фильтр-вкладки скроллятся, активная видна.
4. Страница фермера: обложка не «съедает» экран, контакты не вылезают за край (длинные ссылки — `word-break`).
5. Нет горизонтального скролла страницы (`overflow-x: hidden` на body как страховка).

## Инструменты проверки
- DevTools → Device Toolbar (iPhone SE 375px, iPad 768px, Desktop 1440px).
- Lighthouse (mobile) — см. `PERFORMANCE.md`.
