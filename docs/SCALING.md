# Масштабирование

Как каталог растёт от 6 записей до сотен — без переписывания.

## Этап 1 — сейчас (до ~30–50 производителей)
- Источник: `data/farmers.json` в репозитории.
- Рендер: клиентский, весь список сразу.
- Добавление: правка JSON + фото. Кода не трогаем.
- Этого достаточно: 50 карточек рендерятся мгновенно.

## Этап 2 — наполнение нетехническим сотрудником
- Источник: **Google Sheets → автоэкспорт в `farmers.json`** (см. `DATA-SOURCE.md`).
- Редактор работает в привычной таблице; GitHub Action валидирует и коммитит JSON.
- Фронтенд не меняется.

## Этап 3 — сотни записей / упор на SEO
- **Статическая генерация:** скрипт на сборке проходит по `farmers.json` и генерирует:
  - `farmer/<id>.html` — реальные страницы с контентом в исходном HTML (отличный SEO);
  - `sitemap.xml`;
  - предотрисованный каталог.
  Это снимает зависимость SEO от JS-рендера.
- **Производительность списка:** пагинация / «показать ещё» / виртуализация (см. `PERFORMANCE.md`).
- **Поиск:** при тысячах записей — индекс на клиенте (например `lunr`/`fuse.js`) или серверный поиск.

## Что заложено для роста
| Возможность | Как масштабируется |
|---|---|
| Категории | Добавляются в `categories[]`; фильтр строится динамически — новые вкладки появляются сами. |
| Производители | Объект в `farmers[]`; URL по `id`. |
| Источник данных | Загрузчик данных изолирован — JSON/Sheets/Airtable меняются в одном месте. |
| Поля карточки | Новые поля добавляются в модель; рендер опциональных блоков уже скрывает пустые. |
| Фото-галереи | `gallery[]` на фермера; блок появляется автоматически. |

## Что НЕ делать преждевременно
- Не вводить SPA-фреймворк / сборку, пока хватает статики.
- Не переходить на Airtable/БД без реальной потребности (вложения, роли, тысячи записей).
- Не добавлять бэкенд — каталог по природе статичен.

## Триггеры перехода между этапами
- **→ Sheets:** контент начал править не-разработчик / правки чаще раза в неделю.
- **→ Статическая генерация:** >100 записей **или** SEO стало бизнес-приоритетом.
- **→ Поисковый индекс/виртуализация:** >300–500 записей или тормоза при фильтрации.
