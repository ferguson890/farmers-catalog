# CLAUDE.md

Контекст проекта для Claude Code и других AI-ассистентов. Прочитай перед любой задачей.

## Что это за проект

Каталог фермеров/производителей **Ассоциации рестораторов Нижегородской области (АРНО)**. Размещается на Tilda (сайт `nireal.ru`). Цель — витрина производителей, с которыми работают рестораны-члены Ассоциации, с возможностью самостоятельного наполнения сотрудниками без разработчика.

- **Функциональный референс:** исходный HTML-прототип (в `design-reference/`).
- **Визуальный референс:** сайт `nireal.ru` — минимализм, чистота, натуральная палитра.
- **Платформа публикации:** Tilda (Zero Block / T123 HTML-блок).

## Принципы (важно соблюдать)

1. **Данные отделены от кода.** Весь контент — в `data/farmers.json`. Никогда не хардкодь производителей в HTML/JS. Добавление фермера = правка JSON, не кода.
2. **Без зависимостей и сборки.** Чистый HTML/CSS/JS (ES5-совместимый, без фреймворков). Должно открываться как статика на Tilda/GitHub Pages. Не вводи npm-сборку без явной необходимости и согласования.
3. **Дизайн = минимализм + воздух + акцент на фото.** Натуральная спокойная палитра (зафиксирована в `:root` в `styles.css`). Не добавляй визуальный шум, тяжёлые тени, лишний декор.
4. **Категории — единый список.** Категории живут в `data/farmers.json → categories`. Поле `category` у фермера обязано совпадать со `slug`. Сейчас: `greens`, `dairy`, `confectionery`, `meat`.
5. **Адаптивность обязательна.** Любая правка верстки проверяется на desktop / tablet / mobile. Брейкпоинты: 1000px, 680px.
6. **Безопасность вывода.** Любой пользовательский/контентный текст экранируется (`esc()` в JS) — не вставляй сырой HTML из данных.

## Карта файлов

| Файл | Назначение |
|------|-----------|
| `index.html` | Каталог. Подключает `src/css/styles.css` и `src/js/catalog.js`. |
| `farmer.html` | Страница производителя, читает `?id=`. Подключает `src/js/farmer.js`. |
| `data/farmers.json` | **Единственный источник данных.** `meta`, `categories[]`, `farmers[]`. |
| `src/css/styles.css` | Общие стили. Палитра — в `:root`. Блоки: NAV, HERO, FILTER, CARD, страница `.fp-*`, FOOTER, RESPONSIVE. |
| `src/js/catalog.js` | Загрузка JSON, рендер табов/карточек, фильтр, поиск. Поддерживает `window.FARMERS_DATA` для инлайна на Tilda. |
| `src/js/farmer.js` | Рендер страницы по `id` из query-строки. |
| `docs/*` | Документация по темам (Tilda, SEO, перф, данные, риски и т.д.). |

## Модель данных (кратко)

Полное описание — `docs/DATA-MODEL.md`. Ключевые поля фермера:
`id` (slug, уникальный, используется в URL), `num`, `icon` (emoji-fallback), `name`, `owner`, `category` (slug), `categoryLabel`, `district`, `address`, `since`, `photo`, `gallery[]`, `shortDesc` (для карточки), `description`, `founderStory`, `products[]`, `productsDetail`, `highlights[]`, `phone`, `phones[]`, `email`, `website`, `social{vk,telegram,other[]}`, `map`, `photosSource`, `status{called,ready}`, `notes`.

## Частые задачи

- **Добавить фермера:** добавить объект в `farmers[]`, фото в `assets/images/<id>/cover.jpg`. Проверить валидность JSON.
- **Добавить категорию:** добавить объект в `categories[]` (slug + label + icon + nav). Затем можно присваивать фермерам.
- **Поменять палитру:** только переменные в `:root` (`styles.css`).
- **Перенести на Tilda:** см. `docs/TILDA.md` (T123-блок + инлайн данных или fetch с CDN).

## Состояние и открытые вопросы

- Фото производителей хранятся как ссылки на Яндекс.Диск (в реестре) — нужно скачать, оптимизировать и положить в `assets/images/<id>/`. Сейчас карточки используют emoji-fallback.
- Требуют уточнения у производителей (см. `notes` в JSON): год основания «Gastro-Ferma», основной телефон и регион (с. Шершово); полный ассортимент и фото для «Городецкие мотивы», «Пряник из Городца», «Сливочная улица».
- Smart Farm физически во Владимирской обл. — по решению заказчика оставлен в каталоге.

## Тон и стиль кода

- Комментарии и UI-тексты — на русском.
- JS — ES5-совместимый, без стрелочных функций в продакшен-коде (для максимальной совместимости со встраиванием на Tilda). IIFE, `var`.
- Не переусложнять. Это статический каталог, а не SPA.
